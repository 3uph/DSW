                                Instalación de Drupal 9


La última versión de Drupal 9 se puede descargar de la URL

https://www.drupal.org/download

La creación de un sitio web nuevo con Drupal 9 consiste en una serie de etapas que se pueden
seguir siempre que se necesita crear un sitio nuevo. En un mismo servidor pueden coexistir
varios sitos de Drupal. Estas fases o etapas son:

1. Crear una base de datos para el sitio web

2. Creación de una instancia de Drupal: descargar Drupal (aunque se puede hacer
manualmente, aquí usaremos el instalador Composer, anteriormente instalado).

3. Configurar el sitio Drupal, incluyendo la conexión con la base de datos



1. Crear una base de datos para el sitio web

En primer lugar resaltar que conviene usar un nombre para la base de datos, un usuario y una
contraseña que recuerdes con facilidad, y que no se olvide. Para facilitar este objetivo vamos a
usar drupal como nombre de usuario y de la base de datos. Aunque luego podemos cambiarlo
(y de hecho hace falta si quieres usar más de una base de datos para más de un sitio web de
Drupal).

# mysql -u root -p

Hacer copias de seguridad en Drupal

Para hacer una copia de un sitio web Drupal se deben seguir varios pasos:

1. Copiar la base de datos

# mkdir ~/backups

# mysqldump -u usuario -p nom_base_datos >
~/backups/nom_base.sql

Enter password: [aquí se utiliza la contraseña del usuario de la
base de datos]

esto realizaría un fichero de backup (nom_base.sql).

2. Copiar los ficheros del directorio donde está el sitio web

Para hacer un backup de los ficheros del directorio del sitio web se puede hacer:

# cd /var/www/html/

# tar zcvf ~/backups/directorio_drupal.tgz directorio_web_drupal

Estos comandos crearían un fichero comprimido con el contenido de los ficheros del sitio web.

Hacer un clón

Se puede hacer un clón de un sitio web usando el backup mostrado en el proceso anterior.
Para ello debemos crear una base de datos nueva para la nueva web (dado que es otra
distinta). Se pueden seguir las instrucciones vistas en este manual, creando la base de datos, el
usuario y la contraseña. A continuación se pueden volcar los contenidos del backup en la
nueva base de datos:

# mysql -u nuevo_usuario -p nueva_base < ~/backsups/nom_base.sql

Enter password: [nueva contraseña]

También es necesario crear un nuevo subdirectorio en /var/www/html con la copia de los
ficheros que se desea clonar. Se puede descomprimir el fichero tar o hacer una copia directa:

# cd /var/www/html

# cp -r directorio_original directorio_clonado

También es necesario editar el fichero de configuración, en su parte final:

/var/www/html/directori_clonat/web/sites/default/settings.php

E introducir los datos de la base de datos clonada (dado que no hacemos el proceso guiado
configuración inicial de drupal).

$databases['default']['default'] = array (

                                               9
'database' => „nueva_base‟,

'username' => „nuevo_usuario',

'password' => „nueva_contrasenya',

'prefix' => '',

'host' => 'localhost',

'port' => '3306',

'namespace' => 'Drupal\\Core\\Database\\Driver\\mysql',

'driver' => 'mysql',

);

Un último paso consistiría en borrar las caches de la página clonada, que se corresponden al
anterior sitio web. Esto se hace ejecutando el comando siguiente en el directorio de la
instalación del clon de Drupal, en nuestro ejemplo sería:
/var/www/html/directorio_clonado/

# vendor/bin/drush cache-rebuild

[success] Cache rebuild complete.

Esto nos permitiría acceder al clon de nuestro web en:

http://localhost/directorio_clonado/web
